# Changelog

Все значимые изменения по лабораторной работе №5 «Аппарат сетей Петри» фиксируются в этом файле.

## [1.0.0] - 2026-04-18

### Added
- добавлен исходный модуль `src/DiningPhilosophers.jl`, реализующий базовую структуру модели сети Петри;
- реализованы сущности сети Петри: позиции, переходы, инцидентная матрица, имена позиций и переходов;
- добавлены функции построения классической сети Петри для задачи обедающих философов;
- добавлены функции построения сети Петри с арбитром для ограничения доступа к ресурсам;
- реализована начальная маркировка сети для обеих моделей;
- добавлена функция `vectorfield` для задания правой части модели;
- реализована функция детерминированного моделирования `simulate_ode`;
- реализована функция стохастического моделирования `simulate_stochastic` на основе последовательного выбора переходов;
- добавлена функция `detect_deadlock` для проверки наличия тупикового состояния;
- добавлена функция `plot_marking_evolution` для визуализации изменения маркировки сети во времени;

- добавлен базовый скрипт `scripts/dining_philosophers.jl`;
- реализован запуск классической сети Петри в обычном формате;
- реализован запуск сети Петри с арбитром в обычном формате;
- добавлено сохранение результатов моделирования в CSV-файлы;
- добавлено сохранение графиков в каталог `plots/`;
- добавлен вывод информации о наличии или отсутствии deadlock;

- добавлена literate-версия первого скрипта `scripts/literate/dining_philosophers.jl`;
- добавлены текстовые пояснения к этапам построения модели, запуску симуляции и сохранению результатов;

### Changed
- уточнена структура каталогов проекта с раздельным использованием `src/`, `scripts/`, `data/`, `plots/` и `image/`;
- скорректирован подход к запуску скриптов: выполнение производится из корня лабораторной работы, а не из каталога `scripts/`;
- уточнён способ активации проекта DrWatson для корректной работы `srcdir()`, `datadir()` и `plotsdir()`;
- изменён подход к параметризованным результатам: число сохраняемых графиков ограничено для сохранения читаемости отчёта;
- переработана организация итоговых сравнительных рисунков: один рисунок включает обе модели, что делает отчёт компактнее;
- адаптировано содержимое отчёта под стиль ранее подготовленных Quarto-отчётов;
- доработана структура презентации, чтобы она отражала ход выполнения лабораторной работы, а не только итоги;
- уточнены формулировки пояснений к рисункам, анимациям и итоговым сравнительным графикам;
- скорректирована библиография под академический формат и подготовлен отдельный `bib`-файл.

### Fixed
- исправлена ошибка сохранения CSV-файлов в неверный путь `scripts/data/` при запуске скрипта не из корня проекта;
- исправлена логика работы с каталогами результатов через DrWatson;
- устранена путаница с расположением папки `src` относительно `scripts`;
- скорректирован способ запуска первого скрипта через `julia --project=. scripts/...` из корня лабораторной работы;
- исправлены YAML-ошибки в заголовке Quarto-файла, связанные с неверными отступами и размещением секции `pdf`;
- устранена проблема автодогрузки LaTeX-пакетов через `tlmgr`, вызывавшая длительные зависания при рендеринге PDF;
- отключена автоматическая установка LaTeX-пакетов в Quarto для более предсказуемой сборки;
- устранена проблема несогласованности движков PDF-сборки;
- исправлены настройки шрифтов в Quarto-файле для корректного отображения основного русского текста;
- устранена проблема вывода русского текста в листингах PDF;
- найден и применён рабочий способ отображения кириллицы в листингах с сохранением подсветки;
- исправлены проблемы с отображением комментариев в коде, которые ранее выводились квадратиками;
- устранены ошибки подключения библиографии;
- исправлена причина пустого списка литературы: добавлено корректное подключение `references.bib` и учтён вывод всех записей;
- устранены предупреждения, связанные с неверной интерпретацией отдельных ключей цитирования;
- исправлены ошибки генерации Word-версии, вызванные отсутствием формата `docx` в конфигурации Quarto;
- уменьшено число лишних файлов, возникающих при параметрических и анимационных запусках.

### Notes
- PDF-отчёт собирается через Quarto с использованием `xelatex`;
- для корректного отображения русских комментариев в листингах используется отдельная настройка LaTeX-преамбулы;
- GIF-анимации корректно подходят для HTML-представления, а в PDF, как правило, отображаются как статичные кадры;
- итоговый набор материалов готов к публикации в репозитории и оформлению release.
