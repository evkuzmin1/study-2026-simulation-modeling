# Changelog

## [v1.0.0] — Лабораторная работа №7

### Добавлено

- Подготовлен проект лабораторной работы №7 по дискретно-событийному моделированию.
- Сформирована рабочая структура проекта с каталогами:
  - `src/` — исходные модули моделей;
  - `scripts/` — скрипты запуска и обработки результатов;
  - `data/` — сохранённые таблицы результатов;
  - `plots/` — построенные графики;
  - `report/` — отчёт, изображения, библиография и материалы для генерации;
  - `presentation/` — презентация по результатам работы.

### Исходные модели

- Добавлен модуль `src/MMCModel.jl`.
- В модуле `MMCModel.jl` реализована модель массового обслуживания M/M/c.
- Для модели M/M/c реализованы:
  - структура параметров `MMCParameters`;
  - генерация входящего потока заявок;
  - моделирование обслуживания несколькими параллельными каналами;
  - моделирование очереди при занятости всех серверов;
  - сбор событий прибытия, начала обслуживания и завершения обслуживания;
  - формирование таблицы заявок;
  - расчёт среднего времени ожидания;
  - расчёт среднего времени обслуживания;
  - расчёт среднего времени пребывания заявки в системе;
  - расчёт максимальной длины очереди;
  - расчёт загрузки каналов обслуживания;
  - аналитические характеристики M/M/c для сравнения с симуляцией.

- Добавлен модуль `src/RossModel.jl`.
- В модуле `RossModel.jl` реализована модель Росса для анализа надёжности системы.
- Для модели Росса реализованы:
  - структура параметров `RossParameters`;
  - моделирование работающих машин;
  - моделирование резервных машин;
  - моделирование отказов;
  - моделирование ремонта;
  - очередь на ремонт;
  - использование нескольких ремонтников;
  - фиксация отказа системы при отсутствии резерва;
  - расчёт времени до отказа;
  - расчёт средней очереди на ремонт;
  - расчёт загрузки ремонтников;
  - серия независимых повторов;
  - аналитическая оценка среднего времени до отказа MTTF.

### Основные результаты

- Для модели M/M/c показано, что увеличение числа серверов снижает:
  - среднее время ожидания;
  - среднее время пребывания заявки в системе;
  - максимальную длину очереди;
  - загрузку одного обслуживающего канала.
- Для модели Росса показано, что увеличение числа ремонтников:
  - повышает среднее время до отказа;
  - уменьшает очередь на ремонт;
  - снижает загрузку одного ремонтника;
  - повышает устойчивость системы.
- Итоговый нормированный анализ показал эффект насыщения:
  - после добавления достаточного количества ресурсов дальнейшее улучшение становится менее выраженным.
